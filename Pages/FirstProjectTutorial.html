First Tutorial

Here's how its going to work. Most of the time there will be a full video of the entire building process that you will be able to follow along with.

Along with that video, below will be screenshots and specific callouts to what exactly is going on.

Either way you should be able to follow along and by the end of this have a simple app that you can run on your phone!

    Setting Everything Up

    First off you will need to download Xcode if you havent done so already. (Download Xcode Here)

    Once you have that open go the menu bar up top and choose File > New > Project.

    For this project we will choose a single view application and keep things simple.

    A new screen allowing you to enter some information. Enter the name of the project "FirstProject" will work. And for the rest of the information you will want to uncheck; Use Core Data, Include Unit Tests, Include UI Tests. We will get to those later but for now we do not need them.

    Great! Now we are all set up. Go to the top left of the window and choose iPhone 7 from the list of devices.

    Next press the playbutton to the left and a virtual iPhone should appear with a blank white screen

Xcode Interface

In the Red on the left we have the file inspector. We can view all the files in our project here.

In the Orange in the center is our workspace. This is where we can edit the file currently selected in our file inspector.

In the Purple the right is our attribute inspector. This gives us details and attributes of what we have selected in our workspace.

On the top is our navigation bar. This lets us run the program, select a simulator type or a connected phone. On the right we have two sets of three buttons. Mess around with these and see how they show and hide certain parts of Xcode.

Adding A Label

Let us first add a label to our view. On attribute inspector on the right scroll through the objects and find a label. Or type in the search field on the bottom of the box.

When you find the label click and drag it into our view. Rename it something like "Hi There!"

Now go to the navigation bar and click run once more.

Awesome! You should now have a label appear on the virtual phone that shows the label!

Connecting The Label

Next we need to add some functionality to the label! Let's connect the label to our code and change it programatically.

First up, go to the file inspector on the left and choose Main.storyboard. Once we are in this file click on the top bar of the virtual screen so that it is selected.

Next, Go to the navigation bar and on the right side there should be an icon that looks like two circles. Click this and a new window showing code should appear next to your Storyboard.

This button shows a "context aware" window. So when you selected that virutal screen it knew to open this file. If it doesn't connect to the right file you can manually change it by selecting where it says Automatic > ViewController.swift

Now hold down the control key, click and drag from the label over to the file. Release when you are inside the view controller class but oustide any of the methods.

A popup should appear allowing you to name the outlet. For this label lets call it "nameLabel". A few options are in the popup but we will not change any of them at this point.

  • • Connection: We can create Outlets and Actions by doing the control drag. Outlets will create a reference to the object where an action is used for buttons and interations.
  • • Object: The object that we are attaching to, in this case we are attaching to our ViewController class
  • • Name: Specify a name so that later on we can reference and change properties on the label
  • • Type: You can change the class of the label here.
  • • Storage: This is a more advanced property telling the computer if an object needs to stay around (strong) or dissapear (weak) during different types of activity

Next up lets change the label programatically when the app runs. To do this we will add the code

  • nameLabel = "Hi Johnny"

inside the "viewDidLoad" method. This is a lifecycle method that will run every time the view is initally loaded in the app. There are other lifecycle methods used at different times but we will get to those later.

Now hit run once again and see the new text on the phone!

If the text is cut off on the phone screen go back to the storyboard and click on the label. Click and drag one of the squares to enlarge the label and run the app again!

Halfway there! Connecting a Button

The hard part is out of the way. Next up we are going to add a single button to change the displayed text on the label!

Adding the button uses all the same techniques that we used for adding the label.

The process of adding objects we will use over and over. Soon it will become muscle memory adding and connecting objects to your views.

  • Find the button object in the attribute inspector on the right.
  • click and drag the button object onto the screen.
  • click the assitant editor button on the top right of the nav bar
  • control click drag the button to the view to create an outlet

Button Actions

At this point we should have connected an outlet from out button to our view. This is great if we need to reference the button. But what about a button press?

For this we will follow the same process we use to connect the label before with one difference. Instead of dragging above the "viewDidLoad" method click and drag below the two methods. A screenshot is below.

Now the popup that appears asking for us to specify a name and other properties, instead of an outlet we want to switch to an action. This will present us with a different set of properties to change.

Create a name for our new method, "buttonPress" will work. And change the type to "touchUpInside" which will allow the trigger of the button to be when the finger is lifted from inside the button.

Hit run and you should see a label an a button on the screen!

Adding functionality

We are so close! Next up we need to add code so that our button can change the labels text.

To do this we will enter the following code inside the action method we just created in the previous step.

  • nameLabel.text = "The Button Works!"

Just like before we are changing the text of the label. But the difference now is that this specific code will not be called until you press on that button.

Complete!

You did it! You made your first app. Even something this simple is incredible.

These little steps that you completed are the building blocks of what goes into making a large application like facebook or instagram. Repeated thousands of times with different objects and connects makes very advanced apps.
